# Cases - User Guide

## What is a Case?
A case is a mortgage application. It tracks the journey from application to disbursement (when the bank releases the money).

---

## What Can I Do?

### View Cases - Kanban Board
- **As a user, I can** see all cases organized by stage in columns
- **As a user, I can** drag and drop cases between stages
- **As a user, I can** see case details at a glance (client name, bank, amount)
- **As a user, I can** filter by bank

### View Cases - Table View
- **As a user, I can** switch between "Active" and "All" tabs
- **As a user, I can** search cases by case ID, client name, or bank
- **As a user, I can** see all case info in rows

### Create New Case
- **As a user, I can** click "+" to create a new case
- **As a user, I can** select an existing client
- **As a user, I can** choose mortgage type (Islamic or Conventional)
- **As a user, I can** choose service type (Assisted or Fully Packaged)
- **As a user, I can** select up to 3 banks to apply with
- **As a user, I can** enter property details (emirate, value, transaction type)
- **As a user, I can** enter loan details (amount, term)

### View Case Details
- **As a user, I can** click on any case to open the details panel
- **As a user, I can** see the current stage and bank info
- **As a user, I can** see loan details (amount, term, property value)
- **As a user, I can** see the client's info
- **As a user, I can** click to view the full client profile

### Edit Case Info
- **As a user, I can** click the pencil icon to edit case details
- **As a user, I can** update loan amount, property value, or bank selection
- **As a user, I can** save or cancel changes

### Manage Bank Forms
- **As a user, I can** click "Documents" tab to see bank forms
- **As a user, I can** see which forms are uploaded, missing, or pending
- **As a user, I can** upload bank application forms
- **As a user, I can** delete uploaded forms

### Advance Stage
- **As a user, I can** click "Advance" to move the case to the next stage
- **As a user, I can** add notes about why we're advancing

### Decline Case
- **As a user, I can** click "Decline" if the bank rejected the application
- **As a user, I can** add the reason for decline

### Withdraw Case
- **As a user, I can** click "Withdraw" if the client wants to cancel
- **As a user, I can** add the reason for withdrawal

### Log Calls
- **As a user, I can** log calls related to this case

### Add Notes
- **As a user, I can** add notes about the case progress

### View Activity History
- **As a user, I can** click "Activity" to see all case activity
- **As a user, I can** see stage changes, calls, notes, and document uploads

## Case Stages (The Journey)

```
Processing → Submitted → Conditionally Approved →
Final Offer Letter → Property Transfer → Disbursed
```

| Stage | What's Happening |
|-------|------------------|
| **Processing** | Preparing the application |
| **Submitted** | Application sent to bank |
| **Conditionally Approved** | Bank gave initial approval with conditions |
| **Final Offer Letter** | Bank issued final terms |
| **Property Transfer** | Transferring property ownership |
| **Disbursed** | Money released - DONE! |

### Terminal Stages (End of Journey)

| Stage | What it Means |
|-------|---------------|
| **Declined** | Bank rejected the application |
| **Withdrawn** | Client cancelled the application |

## Tips
1. Keep bank forms organized and uploaded promptly
2. Add notes when advancing stages to track progress
3. Use the Kanban board for a quick overview of all active cases
4. Drag and drop makes moving cases between stages fast
5. Always add a reason when declining or withdrawing
